## Executive Summary & Primary Purpos
**BirrStream** is a targeted financial web application designed for the Ethiopian market, operating completely within the Ethiopian Birr (ETB) ecosystem. At its core, the platform operates as a **"Task-and-Invest" micro-yield system** that pairs short-term subscription crowd-funding with traffic monetization mechanics.



The website has two primary goals: generating platform liquidity through upfront user investments in yield-bearing VIP packages and driving rapid, viral user acquisition via an aggressive three-tier multi-level marketing (MLM) affiliate network.



---



## Core Functionalities & System Logic



### 1. The VIP Subscription Matrix



Users lock up specific capital amounts for an explicit **7-day expiration cycle**. Each package dictates a fixed daily return, completely exhausting its lifecycle at the end of the week, requiring users to reinvest to maintain active status.



| Package Tier | Purchase Cost | Daily Return Rate | Total 7-Day Yield |

| --- | --- | --- | --- |

| **VIP 1** | 500 Birr | 71.43 Birr | 500.01 Birr |

| **VIP 2** | 3,000 Birr | 428.57 Birr | 2,999.99 Birr |

| **VIP 3** | 12,000 Birr | 1,714.28 Birr | 11,999.96 Birr |

| **VIP 4** | 26,000 Birr | 3,714.28 Birr | 25,999.96 Birr |

| **VIP 5** | 50,000 Birr | 7,142.85 Birr | 49,999.95 Birr |

| **Premium Tiers** | *Locked* | *Elite, Apex, Titan, Alpha milestones* | *Requires Tier Unlock* |



### 2. Financial Safety Valves (The 40% Reserve Rule)



To prevent sudden platform insolvency and manage asset flight, the backend enforces a mandatory permanent reserve policy:



> **The 40% Liquidity Reserve Guardrail:** A user's account balance cannot be drawn below **40% of their highest active investment package base value**. For example, a user running a VIP 1 package (500 Birr) must maintain a permanent floor balance of at least 200 Birr inside the ledger; any withdrawal request breaching this constraint is automatically rejected by the system validation engine.



### 3. Traffic & Engagement Loops



* **Daily Tasks Feed:** A traffic steering mechanism that resets nightly at midnight. Users earn direct micro-payouts by executing specific promotional actions, such as streaming music videos (+4 Birr), opening partner landing pages (+2 Birr), or joining specialized Telegram broadcast feeds (+25 Birr).

* **Consecutive Login Streaks:** A gamified 7-day retention calendar that rewards users with an escalating 5 Birr bonus for every consecutive day they log into the interface.

## Comprehensive Page Inventory



### 1. Authentication & Onboarding Gateways



* **Sign-In Portal (`/login`):** Validates access credentials using a clean twin-input schema (Username/Email and Password). Includes session memory tokens ("Remember Me") and entry pathways to password recovery options.

* **Registration Gateway (`/register`):** Handles profile configuration through standard fields (Full Name, Username, Email, and dual Password confirmation fields). Includes an optional **Referral Code Input** that automatically wires the user into an upstream partner's downline network when utilizing shared invite anchors.

* **Credentials Recovery Mode (`/forgot-password`):** A single-field entry layout that verifies user emails to dispatch automated, secure password reset tokens.



### 2. Main Dashboard Hub (`/dashboard`)



Serves as the primary viewport for authenticated user sessions.



* **Asset Metrics Panel:** A series of visual blocks displaying real-time financial variables: `Main Balance`, `Total Yield Cumulative`, `Total Deposited Funds`, and `Total Withdrawn Cash`.

* **Progress Indicators:** A linear status track showcasing the user's distance from unlocking higher-tier VIP assets.

* **Retention Widgets:** Integrates the active 7-day login calendar and primary access links to launch the daily tasks feed.



### 3. VIP Procurement Matrix (`/packages`)



An interactive layout showcasing available subscription tiers.



* Selecting a package triggers an instant balance check. If the account balance is lower than the upfront cost, it launches an error layout: `"Insufficient balance. Need X more Birr to unlock."` If the balance check passes, it activates the package, sets up the daily task returns, and adjusts progress trackers toward the locked premium tiers (Elite, Apex, Titan, Alpha).



### 4. Deposit Gateway Engine (`/deposit`)



Manages the inbound funding stream.



* **Gateway Interface:** Isolates the primary processing rail—specifically tailored around **Telebirr (Ethio Telecom)** integrations.

* **Account Verification Instructions:** Displays clear text instructions detailing the exact manual transfer parameters, including the designated corporate recipient wallet details (e.g., Target ID Name: `MUHIDDIN`).

* **Verification Upload Terminal:** A form where the user inputs the exact deposit amount (ranging from 500 to 50,000 Birr) and the sender's full name, accompanied by a file upload input (`Tap to upload receipt`) to submit proof-of-transfer screenshots for admin validation.



### 5. Withdrawal Engine & Bank Configurations (`/withdraw`)



Handles outward user cash requests.



* **Bank Parameter Matrix (`/withdrawal-settings`):** A locked configuration form where users securely map out their permanent cashout routes, including Bank/Wallet Type selection, Account Name strings, and unique Wallet ID numbers.

* **Payout Request Terminal:** A simple input screen where users specify their payout amount. Submitting this screen runs an immediate automated validation check against the active **40% permanent reserve constraint** before passing the ticket to the pending administrative payouts queue.



### 6. Growth Engine & Network Portals



* **Referral Generation Terminal (`/referral`):** A page featuring tools designed for fast link distribution. It provides the user with an explicit alphanumerical invite string, an auto-routing registration hyperlink, and an automatically generated QR code image for print or social media sharing.

* **Affiliate Network Center (`/affiliate-network`):** A dashboard monitoring downline user generation growth across a 3-tier matrix. Includes tabs to filter performance data by downline generation:

* *Level 1:* Direct invitees.

* *Level 2:* Secondary generation users brought in by Level 1 affiliates.

* *Level 3:* Generational downstream users brought in by Level 2 affiliates.

* *Downstream Detailed Ledger:* A table showing individual invite performance data, displaying registration timestamps, active deposit footprints, and calculated upstream commission payouts.





* **VIP Upgrades Hub (`/vip-upgrades`):** An alternative progression path allowing users to bypass traditional cash purchase requirements. It displays performance milestone goals (e.g., minimum active direct referrals or aggregate downline deposit turnover metrics) that automatically grant premium VIP access statuses upon completion.



### 7. Core Ledger & Utilities



* **Central Transaction Ledger (`/transactions`):** A comprehensive historical system log where users can sort account actions using quick filter tags: `All`, `Deposits`, `Withdrawals`, `Task Earnings`, and `Commissions`.

* **AI Support Hub (`/support`):** An automated modal overlay chat window featuring direct quick-reply macro command buttons (e.g., *"Why is my withdrawal pending?"*, *"How to complete daily tasks?"*) and a standard text/attachment input field to escalate tickets directly to support operators.

* **Account Removal Form (`/delete-account`):** A secure, high-risk profile deletion layout requiring users to enter matching string challenges to erase data profiles from active platform servers.
